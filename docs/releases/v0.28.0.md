# modelmux v0.28.0

## CLI Power-Up Release

This release brings full-featured CLI subcommands for dispatch, broadcast, and feedback — making modelmux a first-class command-line tool for scripts, CI pipelines, and interactive use.

### New CLI Commands

- **`modelmux broadcast`** — Parallel multi-provider dispatch with JSON output
  - `--providers codex gemini` to target specific providers
  - `--compare` for structured similarity analysis (agreement score, speed ranking, unique terms)
- **`modelmux feedback`** — Submit or view user ratings for routing improvement
  - `modelmux feedback --run-id abc --provider codex --rating 5`
  - `modelmux feedback --list` to view recent entries
- **`modelmux dispatch --failover`** — Auto-retry with other providers on failure
- **`modelmux dispatch --max-retries N`** — Same-provider retry with exponential backoff (2s, 4s, 8s...)
- **`modelmux check --json`** — Structured JSON output for CI
- **`modelmux history --source`** — Filter by dispatch source (dispatch, broadcast, cli-dispatch, cli-broadcast)

### Dashboard Real-Time Updates

- **SSE endpoint** (`/api/events`): Pushes active dispatches, stats, and history every 2 seconds
- Frontend auto-connects to SSE with polling fallback for older browsers
- Status indicator shows "live (SSE)" or "auto-refresh: 5s (polling)"

### MCP Tool Improvements

- **`mux_dispatch` retry**: New `max_retries` parameter (1-5) with exponential backoff before failover
- History entries now track source type (MCP dispatch, CLI dispatch, broadcast, collaboration)

### Stats

- 459 tests passing (+18 new)
- 11 CLI subcommands: `dispatch`, `broadcast`, `feedback`, `a2a-server`, `dashboard`, `init`, `config`, `check`, `status`, `history`, `version`

# modelmux v0.28.0

## CLI 全面增强版

本版本为 dispatch、broadcast、feedback 提供了完整的 CLI 子命令支持，使 modelmux 成为脚本、CI 流水线和交互使用的一等公民命令行工具。

### 新 CLI 命令

- **`modelmux broadcast`** — 并行多 provider 分发，JSON 输出
  - `--providers codex gemini` 指定目标 provider
  - `--compare` 结构化对比分析（一致性评分、速度排名、独特词汇）
- **`modelmux feedback`** — 提交或查看用户评分
  - `modelmux feedback --run-id abc --provider codex --rating 5`
  - `modelmux feedback --list` 查看最近条目
- **`modelmux dispatch --failover`** — 失败时自动切换到其他 provider
- **`modelmux dispatch --max-retries N`** — 同 provider 指数退避重试
- **`modelmux check --json`** — 结构化 JSON 输出（适合 CI）
- **`modelmux history --source`** — 按来源过滤

### Dashboard 实时更新

- **SSE 端点** (`/api/events`)：每 2 秒推送活跃分发、统计和历史
- 前端自动连接 SSE，不支持时回退到轮询
- 状态指示器显示 "live (SSE)" 或 "auto-refresh: 5s (polling)"

### MCP 工具改进

- **`mux_dispatch` 重试**：新增 `max_retries` 参数（1-5），指数退避后再 failover
- 历史记录现在跟踪来源类型

### 统计

- 459 个测试通过（+18 新增）
