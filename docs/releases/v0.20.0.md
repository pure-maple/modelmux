## modelmux v0.20.0

Structured release notes and CI automation improvements.

结构化 Release Notes 和 CI 自动化改进。

### Highlights / 亮点

- **Structured Release Notes / 结构化发布说明**: Each version now has a dedicated bilingual (EN/CN) release notes file in `docs/releases/`.
  每个版本现有独立的中英双语 Release Notes 文件，存放在 `docs/releases/` 目录下。

- **Auto GitHub Release / 自动 GitHub Release**: CI now automatically creates GitHub Releases from `docs/releases/{tag}.md` on tag push.
  CI 现在在推送 tag 时自动从 `docs/releases/{tag}.md` 创建 GitHub Release。

- **Release History / 历史补全**: Backfilled release notes for all versions v0.1.0 through v0.19.0.
  补全了 v0.1.0 至 v0.19.0 所有版本的 Release Notes。

- **TaskStore JSONL Persistence / 任务持久化**: Completed/failed/canceled A2A tasks persisted to `~/.config/modelmux/a2a-tasks.jsonl`, restored on server restart.
  已完成/失败/取消的 A2A 任务持久化到 `~/.config/modelmux/a2a-tasks.jsonl`，服务器重启时自动恢复。

- **A2A E2E Tests / A2A 端到端测试**: 10 async integration tests using `httpx.AsyncClient` + ASGI transport (no network needed).
  10 个异步集成测试，使用 `httpx.AsyncClient` + ASGI 传输层（无需网络连接）。

- **`--no-persist` CLI Flag / CLI 参数**: Disable A2A task persistence for in-memory-only mode.
  禁用 A2A 任务持久化，启用纯内存模式。

### CI/CD Pipeline / CI/CD 流水线

```
git tag v0.20.0 && git push origin v0.20.0
  → CI: test → build → PyPI publish → GitHub Release (auto)
```

The release job reads from `docs/releases/v0.20.0.md` — no manual release creation needed.

每次打 tag 推送后自动完成测试、构建、PyPI 发布、GitHub Release 创建。Release 任务从 `docs/releases/v0.20.0.md` 读取内容，无需手动创建。

### Release Notes Workflow / 发版流程

1. Write `docs/releases/vX.Y.Z.md` with bilingual content.
   编写包含中英双语内容的 `docs/releases/vX.Y.Z.md` 文件。
2. Bump version in `__init__.py`, `pyproject.toml`, `server.json`.
   在 `__init__.py`、`pyproject.toml`、`server.json` 中更新版本号。
3. Commit, tag, push — CI handles the rest.
   提交、打 tag、推送——CI 自动完成后续流程。
