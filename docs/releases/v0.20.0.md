## modelmux v0.20.0

Structured release notes and CI automation improvements.

### Highlights

- **Structured Release Notes**: Each version now has a dedicated bilingual release notes file in `docs/releases/`.
- **Auto GitHub Release**: CI now automatically creates GitHub Releases from `docs/releases/{tag}.md` on tag push.
- **Release History**: Backfilled release notes for all versions v0.1.0 through v0.19.0.
- **TaskStore JSONL Persistence**: Completed/failed/canceled A2A tasks persisted to `~/.config/modelmux/a2a-tasks.jsonl`, restored on server restart.
- **A2A E2E Tests**: 10 async integration tests using `httpx.AsyncClient` + ASGI transport (no network needed).
- **`--no-persist` CLI Flag**: Disable A2A task persistence for in-memory-only mode.

### CI/CD Pipeline

```
git tag v0.20.0 && git push origin v0.20.0
  -> CI: test -> build -> PyPI publish -> GitHub Release (auto)
```

The release job reads from `docs/releases/v0.20.0.md` — no manual release creation needed.

### Release Notes Workflow

1. Write `docs/releases/vX.Y.Z.md` with bilingual content.
2. Bump version in `__init__.py`, `pyproject.toml`, `server.json`.
3. Commit, tag, push — CI handles the rest.

### Test Coverage

146 tests passing (9 new persistence tests + 10 E2E tests)

```bash
uvx modelmux
# or
pip install modelmux==0.20.0
```

## modelmux v0.20.0

结构化 Release Notes 和 CI 自动化改进。

### 亮点

- **结构化发布说明**：每个版本现有独立的中英双语 Release Notes 文件，存放在 `docs/releases/` 目录下。
- **自动 GitHub Release**：CI 现在在推送 tag 时自动从 `docs/releases/{tag}.md` 创建 GitHub Release。
- **历史补全**：补全了 v0.1.0 至 v0.19.0 所有版本的 Release Notes。
- **任务持久化**：已完成/失败/取消的 A2A 任务持久化到 `~/.config/modelmux/a2a-tasks.jsonl`，服务器重启时自动恢复。
- **A2A 端到端测试**：10 个异步集成测试，使用 `httpx.AsyncClient` + ASGI 传输层（无需网络连接）。
- **`--no-persist` CLI 参数**：禁用 A2A 任务持久化，启用纯内存模式。

### CI/CD 流水线

```
git tag v0.20.0 && git push origin v0.20.0
  -> CI: test -> build -> PyPI publish -> GitHub Release (auto)
```

每次打 tag 推送后自动完成测试、构建、PyPI 发布、GitHub Release 创建。Release 任务从 `docs/releases/v0.20.0.md` 读取内容，无需手动创建。

### 发版流程

1. 编写包含中英双语内容的 `docs/releases/vX.Y.Z.md` 文件。
2. 在 `__init__.py`、`pyproject.toml`、`server.json` 中更新版本号。
3. 提交、打 tag、推送——CI 自动完成后续流程。

### 测试覆盖

共 146 个测试通过（9 个新增持久化测试 + 10 个端到端测试）
