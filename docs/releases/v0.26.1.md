# modelmux v0.26.1 — Quality & Observability Patch

**Release Date**: 2026-03-07
**Tests**: 417 passed

## Highlights

Post-v0.26.0 quality improvements: silent exception cleanup across the codebase, routing diagnose accuracy fix, dashboard feedback visualization, and standardized release notes workflow.

## Quality & Observability

- **Silent exception cleanup**: Replaced bare `except: pass` with `logger.warning`/`logger.debug` across 7 core modules — disk full, permission errors, and config parse failures are now visible in logs instead of silently swallowed
- **mux_check diagnose parity**: Diagnose mode now checks `routing_rules` before `smart_route()`, matching actual `_auto_route()` logic. Previously, diagnose could report a different provider than what dispatch would actually use.
- **Dashboard feedback panel**: New `/api/feedback` endpoint returns per-provider feedback scores; UI shows star ratings with auto-refresh
- **Import sorting**: Fixed I001 ruff violations in server.py, dashscope.py, http_server.py

## Documentation & Process

- **Release notes workflow**: Dev workflow now **requires** `docs/releases/vX.Y.Z.md` before tagging. CI reads this file for GitHub Release content. Added `pre-release-check.sh` to validate before publishing.
- **Backfilled release notes**: v0.25.1 and v0.26.0 GitHub releases updated with full bilingual content
- **ROADMAP update**: Reflects current state — routing v4, feedback loop, security hardening phases
