## modelmux v0.18.0

A2A Collaboration Engine — true multi-agent iterative collaboration.

### Highlights

- **mux_collaborate**: New MCP tool for iterative multi-agent collaboration with convergence detection.
- **3 Collaboration Patterns**:
  - **Review**: Implement -> Review -> Revise loop until quality criteria met.
  - **Consensus**: Multiple perspectives in parallel -> Synthesis -> Agreement check.
  - **Debate**: Adversarial argumentation -> Rebuttal -> Arbiter judgment.
- **A2A Data Model**: Task state machine (`submitted -> working -> completed/failed`), Message/Part/Artifact, AgentCard.
- **Layered Context**: Pinned facts + rolling summary + recent window + artifact index.
- **4-Tier Convergence**: Hard limit -> Structured signals -> Stability detection -> LLM judge.
- **Transport Independent**: Engine can serve both MCP and A2A HTTP simultaneously.

### Architecture

```
mux_collaborate(pattern="review", task="...")
  +-- CollaborationEngine
       +-- Context Manager (layered memory)
       +-- Convergence Detector (4 tiers)
       +-- Pattern Executor
            +-- review:    implementer <-> reviewer
            +-- consensus: analyst x N -> synthesizer
            +-- debate:    proponent <-> opponent -> arbiter
```

## modelmux v0.18.0

A2A 协作引擎 — 真正的多 Agent 迭代协作。

### 亮点

- **mux_collaborate 协作工具**：全新 MCP 工具，支持迭代式多 Agent 协作与收敛检测。
- **三种协作模式**：
  - **审查**：实现 -> 审查 -> 修订循环，直到满足质量标准。
  - **共识**：多视角并行分析 -> 综合整理 -> 一致性检查。
  - **辩论**：对抗式论证 -> 反驳 -> 仲裁裁决。
- **A2A 数据模型**：任务状态机（`submitted -> working -> completed/failed`）、Message/Part/Artifact、AgentCard 完整数据模型。
- **分层上下文**：固定事实 + 滚动摘要 + 近期窗口 + 产出物索引，四层上下文管理。
- **四层收敛检测**：硬限制 -> 结构化信号 -> 稳定性检测 -> LLM 裁判，四层收敛判定机制。
- **传输层无关**：引擎可同时服务于 MCP 和 A2A HTTP 两种传输层。

### 架构

```
mux_collaborate(pattern="review", task="...")
  +-- CollaborationEngine
       +-- Context Manager (layered memory)
       +-- Convergence Detector (4 tiers)
       +-- Pattern Executor
            +-- review:    implementer <-> reviewer
            +-- consensus: analyst x N -> synthesizer
            +-- debate:    proponent <-> opponent -> arbiter
```
