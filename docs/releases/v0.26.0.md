# modelmux v0.26.0 — User Feedback Loop & Routing v4

**Release Date**: 2026-03-07
**Tests**: 413 passed

## Highlights

Introduces user feedback as a first-class routing signal. The new `mux_feedback` tool lets users rate dispatch results, and routing v4 incorporates these ratings into provider selection alongside keyword patterns, dispatch history, and benchmark scores.

## New Features

### `mux_feedback` Tool
- Rate dispatch results on a 1-5 scale to improve future routing decisions
- Auto-detects provider and task category from recent dispatch history
- `list_recent=True` to review your recent feedback entries
- Persistent storage: `~/.config/modelmux/feedback.jsonl`

### Routing v4: Four-Signal Composite Scoring
- **Keyword patterns** (35%): Task category classification via keyword matching
- **Dispatch history** (25%): Success rate and latency from past dispatches
- **Benchmark quality** (20%): Standardized task scoring from benchmark suite
- **User feedback** (20%): Aggregated user ratings per provider per category
- Adaptive weight degradation: when a data source is unavailable, its weight is redistributed proportionally to remaining signals
- `mux_check` diagnostics now show the full v4 routing breakdown

## Bug Fixes (from DashScope cross-review)

### P0 — Critical
- **Engine convergence fix**: `_execute_round` sequential mode now correctly returns turns instead of `[]`, enabling `convergence.evaluate()` to detect convergence. Removed duplicate `ctx.update_after_turn` call that caused double context updates.
- **Generator return value fix**: `BaseAdapter.run()` now uses `while`/`next` loop instead of `for` to correctly capture `stream_subprocess` exit code via `StopIteration.value`.

### P1 — Important
- **History metadata fix**: `log_result` now places metadata (`ts`/`source`/`task`) after `**result_dict` spread, preventing adapter results from overriding system-authoritative fields.
- **Audit timezone fix**: `read_recent` treats naive ISO timestamps as UTC to match `time.time()` cutoff comparison.

## Test Coverage

25 new tests:
- 14 feedback tests (scoring, persistence, routing integration)
- 11 bugfix regression tests (convergence, generator, metadata, timezone)

# modelmux v0.26.0 — 用户反馈循环 & 路由 v4

**发布日期**: 2026-03-07
**测试**: 413 通过

## 核心变更

引入用户反馈作为一等路由信号。新增 `mux_feedback` 工具让用户对分发结果评分，路由 v4 将评分纳入 provider 选择的四维复合评分。

## 新功能

- **`mux_feedback` 工具**: 1-5 分评分，自动关联 provider 和任务类别
- **路由 v4**: 关键词 (35%) + 历史 (25%) + 基准测试 (20%) + 用户反馈 (20%)
- 自适应权重退化：数据源不可用时自动重分配权重

## Bug 修复

- **协作引擎收敛修复** (P0): sequential 模式正确返回 turns，修复双重上下文更新
- **Generator 返回值修复** (P0): 正确捕获 subprocess 退出码
- **History 元数据修复** (P1): 系统字段不可被适配器结果覆盖
- **审计时区修复** (P1): naive 时间戳统一视为 UTC
