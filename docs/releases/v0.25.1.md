# modelmux v0.25.1 — Security Hardening Phase 2

**Release Date**: 2026-03-07
**Tests**: 388 passed

## Highlights

Completes the security hardening work started in v0.25.0, addressing all remaining findings from the 3-agent security audit. Focus areas: policy enforcement parity across all MCP tools, A2A network exposure, and input validation hardening.

## Security Fixes

### Policy Enforcement Parity
- **`mux_broadcast`**: Now checks policy for ALL target providers (previously only checked the first)
- **`mux_workflow`**: Enforces policy before execution
- **`mux_collaborate`**: Enforces policy before execution

### A2A Server Hardening
- **Default Binding**: A2A server now defaults to `127.0.0.1` instead of `0.0.0.0` — prevents unintended network exposure. Use `--host 0.0.0.0` to explicitly opt-in.
- **Request Body Size Limit**: 1MB max request body to prevent DoS
- **Task ID Hijacking Prevention**: `tasks/send` and `tasks/sendSubscribe` generate server-side task IDs, ignoring client-supplied IDs to prevent cross-client task manipulation
- **Timeout Cap**: `timeout_per_turn` capped at 3600s to prevent resource exhaustion

### Adapter Security
- **GenericAdapter Template Injection**: Built-in substitution keys (`task`, `workdir`, `sandbox`, `session_id`) protected from `extra_args` override
- **Environment Variable Blocklist**: `ProviderConfig.to_env_overrides()` blocks dangerous env vars (`PATH`, `LD_PRELOAD`, `PYTHONPATH`, etc.)
- **DashScope SSRF Fix**: `base_url` no longer accepted from `extra_args` (only from config/env overrides)

### Observability
- **Policy Parse Logging**: Failed `policy.json` parsing now logs a warning instead of silently falling back to permissive defaults

## Test Coverage

15 new security tests covering policy enforcement parity, binding defaults, request size limits, task ID generation, timeout caps, template injection, env var blocklist, and SSRF prevention.

# modelmux v0.25.1 — 安全加固 Phase 2

**发布日期**: 2026-03-07
**测试**: 388 通过

## 核心变更

完成 v0.25.0 开始的安全加固工作，修复 3-agent 安全审计的所有剩余发现。

## 安全修复

- **策略执行一致性**: `mux_broadcast`/`mux_workflow`/`mux_collaborate` 现在全部执行策略检查
- **A2A 默认绑定**: 默认 `127.0.0.1`，防止意外网络暴露
- **请求体大小限制**: 1MB 上限防 DoS
- **Task ID 劫持防护**: 服务端生成 ID，忽略客户端提供的 ID
- **超时上限**: `timeout_per_turn` 上限 3600s
- **模板注入防护**: GenericAdapter 内置变量不可被 extra_args 覆盖
- **环境变量黑名单**: 阻止 PATH/LD_PRELOAD 等危险环境变量
- **DashScope SSRF**: base_url 仅从配置/环境变量读取
- **策略解析日志**: 失败时记录警告而非静默回退
