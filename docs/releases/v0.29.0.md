# modelmux v0.29.0

## Quality & Coverage Release

This release focuses on test coverage, code quality, and the new `modelmux clean` command. Coverage jumps from ~50% to 83% with 937 tests across all modules.

### New Features

- **`modelmux clean`** — Remove data files (history, audit, feedback, status, benchmark)
  - `modelmux clean` removes all data files
  - `modelmux clean history` targets a specific type
  - `--dry-run` previews what would be removed without deleting
- **`modelmux history --json`** — JSON output for scripting and CI integration
- **DashScope in init wizard** — `modelmux init` now detects and configures DashScope API
- **`modelmux check` enhanced** — Shows config profiles, routing rules, and 24h history summary
- **`modelmux export --source`** — Filter exports by dispatch source

### Bug Fixes

- **Dashboard `api_providers`**: Uses `check_available()` instead of binary check for DashScope support
- **Benchmark async**: Fixed async runner for provider benchmarks

### Test Coverage

- **937 tests** passing (from 459 in v0.28.0)
- **83% overall coverage** (from ~50%)
- Key module improvements:
  - server.py: 37% → 76%
  - cli.py: 40% → 65%
  - init_wizard.py: 28% → 87%
  - convergence.py: → 100%
  - context.py: → 100%
  - workflow.py: → 100%
  - compare.py: → 100%
  - adapters (codex/gemini/claude): → 97-100%

### Stats

- 937 tests, 83% coverage
- 12 CLI subcommands: `dispatch`, `broadcast`, `feedback`, `clean`, `a2a-server`, `dashboard`, `init`, `config`, `check`, `status`, `history`, `version`
