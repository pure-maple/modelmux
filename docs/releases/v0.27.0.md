# v0.27.0 — CLI Dispatch & GitHub Actions

## New Features / 新功能

### `modelmux dispatch` CLI subcommand
Run a single task from the command line with JSON output. Supports smart routing, provider/model override, configurable timeout and sandbox level.

新增 `dispatch` CLI 子命令，可从命令行直接运行任务并输出 JSON。支持智能路由、provider/model 指定、超时和沙箱配置。

```bash
# Auto-route to best available provider
modelmux dispatch "Review this code for bugs"

# Specify provider and model
modelmux dispatch -p codex -m gpt-5.4 "Analyze architecture"

# Pipe from stdin
cat diff.txt | modelmux dispatch -p gemini --timeout 600
```

### GitHub Actions Composite Action
Reusable action at `.github/actions/review/` for automated PR code review. Extracts PR diff, dispatches to any provider via `modelmux dispatch`, and posts results as PR comments.

新增 GitHub Actions 可复用 action，用于 PR 自动代码审查。自动提取 diff → dispatch review → 发 PR 评论。

```yaml
- uses: pure-maple/modelmux/.github/actions/review@main
  with:
    provider: auto
```

## Improvements / 改进

- **Routing data TTL cache**: 60-second in-memory cache for history/benchmark/feedback with auto-invalidation on writes
- **Provider health in `mux_check`**: `last_used_ago`, `avg_latency`, `success_rate` per provider
- **Config validation**: Warns on unknown top-level keys (catches typos in profiles.toml)
- **Routing v4 test coverage**: 90% → 93%

路由数据 TTL 缓存（60s）、provider 健康度摘要、配置未知键警告、路由 v4 测试覆盖率提升。
