## modelmux v0.15.0

Custom provider plugins — bring any CLI tool as a provider.

### Highlights

- **Custom Providers**: Register any CLI tool as a modelmux provider via `profiles.toml`.
- **GenericAdapter**: Template-based command execution with `{task}` and `{workdir}` placeholders.
- **Unlimited Providers**: `mux_dispatch` now accepts any registered provider name, not just the built-in 4.
- **mux_check Enhancement**: Custom providers shown with `custom: true` flag.

### Configuration

```toml
# ~/.config/modelmux/profiles.toml
[providers.mybot]
command = "mybot"
args = ["--prompt", "{task}", "--dir", "{workdir}"]
```

```
mux_dispatch(provider="mybot", task="do something")
```

### Install

```bash
uvx modelmux
```

## modelmux v0.15.0

自定义 Provider 插件 — 将任意 CLI 工具注册为 Provider。

### 亮点

- **自定义 Provider**：通过 `profiles.toml` 将任意 CLI 工具注册为 modelmux Provider。
- **通用适配器**：基于模板的命令执行，支持 `{task}` 和 `{workdir}` 占位符。
- **不限 Provider**：`mux_dispatch` 现在接受任意已注册的 Provider 名称，不再局限于内置的 4 个。
- **mux_check 增强**：自定义 Provider 会以 `custom: true` 标记显示。

### 配置

```toml
# ~/.config/modelmux/profiles.toml
[providers.mybot]
command = "mybot"
args = ["--prompt", "{task}", "--dir", "{workdir}"]
```

```
mux_dispatch(provider="mybot", task="do something")
```

### 安装

```bash
uvx modelmux
```
