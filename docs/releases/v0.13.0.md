## modelmux v0.13.0

Workflow templates -- multi-step pipeline orchestration.

### Highlights

- **`mux_workflow`**: New MCP tool for multi-step pipeline execution across providers.
- **Built-in Workflows**: `review` (codex implements, then claude reviews) and `consensus` (multi-model synthesis).
- **Custom Workflows**: Define your own pipelines in `profiles.toml` under `[workflows.*]`.
- **Output Chaining**: Steps reference previous outputs via `{step_name}` placeholders.

### Example

```
mux_workflow(workflow="review", task="Implement user auth module")
→ Step 1: codex implements the code
→ Step 2: claude reviews the implementation
→ Final: combined result with implementation + review feedback
```

### Install

```bash
uvx modelmux
# or
pip install modelmux==0.13.0
```

## modelmux v0.13.0

工作流模板 -- 多步流水线编排。

### 亮点

- **`mux_workflow` 工作流工具**：新增 MCP 工具，支持跨 provider 的多步流水线执行。
- **内置工作流**：内置 `review`（codex 实现 -> claude 审查）和 `consensus`（多模型综合）两种工作流。
- **自定义工作流**：在 `profiles.toml` 的 `[workflows.*]` 中定义自定义流水线。
- **输出链式传递**：步骤间通过 `{step_name}` 占位符引用上游输出，实现数据链式传递。

### 示例

```
mux_workflow(workflow="review", task="Implement user auth module")
→ 第 1 步：codex 实现代码
→ 第 2 步：claude 审查实现
→ 最终：合并实现代码与审查反馈
```

### 安装

```bash
uvx modelmux
# 或
pip install modelmux==0.13.0
```
